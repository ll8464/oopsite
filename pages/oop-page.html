Object-Oriented Programming

Abstraction

Abstraction means to hide away. In OOP, abstraction is used to hide away code and logic. The user should be only be shown mandatory details. For example, when ordering food at a drive-thru, the user only needs to decide their menu item and payment option. They do not need to know how the food is prepared or how many workers are in the restaurant. The business logic is abstracted away from them. This is helpful to developers as well. Instead of rewriting entire blocks of code each time you need a peculiar bit of functionality, you can simply invoke the class or function.

Encapsulation

Encapsulation, also known as data hiding, is the act of enclosing data in one unit. Access to the data is regulated through the use of setters and getters.

Inheritance

Inheritance is the process of passing properties from a parent to a child class.

Polymorphism

Polymorphism allows OOPs to differientiate between entities with the same name. When a child class overwrites a method, it is doing so through the use of polymorphism.
