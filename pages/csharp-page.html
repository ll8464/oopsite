IEnumerable

IEnumerable allows types to be iterable. Every data structure that can be iterated through implements the IEnumerable interface. When creating your own types, you must implement both the IEnumerable and IEnumerator interfaces if you wish to utilize iterable methods (or use foreach).

Before we get into the details about implementation, let's answer an important question: "Why use the IEnumerable interface at all?". It is recommended to use the IEnumerable interface when you donâ€™t want to copy a massive amount of data into memory like in the case of a large database. It is not recommended to use IEnumerable when you need results quickly and if you are editing or changing the objects later on. Then it is better to use an Array/List or other data structure.

Generic IEnumerable <T> vs. Non-Generic IEnumerable

Generic in this context means "relating to a class or group of things". Thus, non-generic means any type is allowed, while generic means only a specific type is allowed. Non-Generic IEnumerable is not strongly typed and store elements as objects. Generic IEnumerable <T> is strongly typed and store elements as a collection/array of the chosen type. Both allows you to iterate through their respective collection using GetEnumerator(). In general, it is better to use generic IEnumerable <T> if you know what type of data that will be returned.

IEnumerable <T> Generic

Generic IEnumerable <T> implements both Non-Generic IEnumerable and IDisposable interfaces. The reason why it is preferred to to use generic IEnumerable is because it is faster when going through large datasets. This is due to its implementation of the Dispose method from IDisposable, which will free resources at each iteration. After 100,000+ iterations in a dataset, even the most modern processors will be noticably slower if IDisposable is not used.
